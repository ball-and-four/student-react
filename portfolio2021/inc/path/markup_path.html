방식
  • - develop(develop은 항상 최신상태 유지) 기준으로 feature 브랜치 생성(feature/#이슈번호_브랜치명)
  • - merge는 pr로 진행(#54 asset 커밋 참고)
규칙
  • - https는 http로 작성
  • - 모든 경로는 /로 작성 (ex. /img, /js, /css)
정의
  • - resource/template: HTML 파일만 사용 (thymeleaf에서 사용하는 기본 템플릿 경로)
  • - resource/static: 정적 파일만 사용 (기본 정적파일 경로)
    resources/static/css
    resources/static/img
    resources/static/js
